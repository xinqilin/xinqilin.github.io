Docker Compose V2 遷移指南

2024年01月15日 by Bill.Lin 4 分鐘閱讀
Container

V1 vs V2 的本質差異

V1V2
執行檔docker-compose（Python）docker compose（Go，Docker CLI plugin）
安裝方式獨立 binary，另行安裝Docker Desktop 內建；Linux 用 docker compose plugin
version: 欄位必填（決定語法版本）已廢棄，出現時忽略並發出警告
網路命名<project>_default（底線）<project>-default（連字號）
啟動指令docker-compose updocker compose up

V1 已於 2023 年 7 月正式 EOL，不再維護。

移除 version: 欄位

V1 時代的 version: "3.9" 決定 Compose 文件語法版本。V2 不再有版本差異，統一支援最新語法。

# V1 寫法（過時，V2 會警告）
version: "3.9"
services:
  app:
    image: my-app:latest

# V2 正確寫法（刪除 version 欄位）
services:
  app:
    image: my-app:latest

移除後不影響任何功能，僅消除 the attribute version is obsolete 警告。

depends_on 加上健康檢查

V1 的 depends_on 只等 container 啟動，不等服務就緒：

2

# V1（不可靠）：MySQL container 啟動了，但 MySQL 程序可能還在初始化
depends_on:
  - db

services:
  app:
    build: .
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_healthy

  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: mydb
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-u", "root", "-proot"]
      interval: 5s
      timeout: 3s
      retries: 10
      start_period: 30s  # 給 MySQL 初始化的緩衝時間

  redis:
    image: redis:7-alpine
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 3s
      timeout: 2s
      retries: 5

condition 值：

condition含義
service_startedcontainer 已啟動（預設，等同 V1 行為）
service_healthyhealthcheck 回報 healthy
service_completed_successfullycontainer 正常退出（code 0），用於初始化容器

docker compose watch（V2.22+）

開發環境熱更新，不需要掛載整個 source volume：

services:
  app:
    build: .
    develop:
      watch:
        - action: sync
          path: ./src
          target: /app/src
          ignore:
            - node_modules/
        - action: rebuild
          path: package.json   # 依賴變更時重建 image
        - action: rebuild
          path: pom.xml

1
2
3
4

# 啟動並開啟 watch 模式
docker compose up --watch
# 或
docker compose watch

action 類型：

action行為
sync將變更的檔案同步進 container（不重啟）
rebuild重新 build image 並重啟 container
sync+restart同步後重啟 container（不重建 image）

# compose.yaml（V2，無 version 欄位）
services:
  app:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "8080:8080"
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://db:3306/mydb
      SPRING_DATASOURCE_USERNAME: app
      SPRING_DATASOURCE_PASSWORD: secret
      SPRING_DATA_REDIS_HOST: redis
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_healthy
    develop:
      watch:
        - action: rebuild
          path: pom.xml
        - action: sync+restart
          path: ./src/main/resources

  db:
    image: mysql:8.0
    environment:
      MYSQL_DATABASE: mydb
      MYSQL_USER: app
      MYSQL_PASSWORD: secret
      MYSQL_ROOT_PASSWORD: rootsecret
    volumes:
      - mysql-data:/var/lib/mysql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-u", "app", "-psecret"]
      interval: 5s
      timeout: 3s
      retries: 10
      start_period: 30s

  redis:
    image: redis:7-alpine
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 3s
      timeout: 2s
      retries: 5

  adminer:
    image: adminer
    ports:
      - "8081:8080"
    profiles: [dev]

volumes:
  mysql-data:

小結

  • 立即行動：移除 version: 欄位、將 docker-compose 替換為 docker compose。
  • depends_on + healthcheck 是防止「DB 還沒好 App 就先啟動」的正確方式。
  • compose watch 讓開發循環更快，適合前後端同時開發的場景。
  • profiles 讓同一份 compose.yaml 服務多個環境，減少維護多份設定的負擔。
