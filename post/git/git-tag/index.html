# 建立 annotated tag（推薦：有作者、日期、訊息）
git tag -a v1.2.0 -m "release: v1.2.0 — JWT security fixes"

# 對特定 commit 打 tag
git tag -a v1.1.0 abc1234 -m "release: v1.1.0"

# 推送單一 tag
git push origin v1.2.0

# 推送所有 local tag
git push origin --tags

# 刪除 local tag
git tag -d v1.2.0

# 刪除 remote tag
git push origin --delete v1.2.0

# 列出所有 tag（依 semver 排序）
git tag -l --sort=-version:refname

# 查看 tag 詳細資訊
git show v1.2.0

Annotated vs Lightweight Tag

AnnotatedLightweight
建立git tag -a v1.0 -m "msg"git tag v1.0
儲存完整 Git object（含 tagger、日期、訊息）只是 commit 的指標
git describe能被找到需加 --tags 才能找到
推薦用途Release、版本里程碑個人臨時標記

GitHub Releases 必須使用 annotated tag（lightweight tag 無法建立 Release）。
